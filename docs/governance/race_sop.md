# 比赛 SOP（race_sop.md）

> 本文件是 CLAUDE.md §4 的补充，专门针对比赛场景的**方案文档 / Demo 录制 / 提交材料**规范。
> 触发时机：每次需要输出方案、写录屏脚本、整理报名附件前必读。

---

## 1. 方案文档规范（Part 1 / Part 2）

### 1.1 字数与结构硬约束

| 字段 | 字数 | 必须覆盖 |
|------|------|---------|
| Part 1 前置分析 | 50-300 字 | 行业数据 + 竞品反馈 + 用户痛点 + 个人理解 |
| Part 2 整体方案 | 300-600 字 | 5 子项：概述/架构/创新/价值/可行性 |

### 1.2 内容自检清单（提交前必过）

#### Part 1 自检 5 项
- [ ] 跳出命题基础信息（不是 copy 命题描述）
- [ ] 引用至少 1 个外部数据（行业报告 / 竞品 / 用户反馈）
- [ ] 体现"我理解这个命题"的深度
- [ ] 字数在 50-300 之间
- [ ] 无错别字 / 无标点错误

#### Part 2 自检 8 项
- [ ] 5 子项全覆盖（概述/架构/创新/价值/可行性）
- [ ] 架构有图（不是纯文字）
- [ ] 创新点 ≥ 3 个，且每个**区别于常规方案**
- [ ] 价值可量化（带数字 / 百分比）
- [ ] 可行性写明"为什么 3 周能出 MVP"
- [ ] 可推广性写明"未来怎么 scale"
- [ ] 字数在 300-600 之间
- [ ] 与 Part 1 风格统一（不矛盾）

### 1.3 风格禁忌

| 禁忌 | 原因 |
|------|------|
| 直接复制命题描述 | 评审一眼看出 |
| 空话"颠覆 / 革命 / 重塑" | 无证据支撑 |
| 大段代码块塞进方案 | 评审没耐心读 |
| 只讲技术不讲业务 | 命题强调"商户成功方法论" |
| 模糊价值"提升效率" | 必须量化"提升 X%" |

## 2. Demo 录制规范（核心 · 评审最爱看）

### 2.1 录屏三件套（缺一不可）

| # | 类型 | 时长 | 内容 |
|---|------|------|------|
| 1 | **3 分钟主 demo** | 180s ± 10s | 完整业务流程（商户咨询 → 6 Agent 协同 → 飞书自动派单） |
| 2 | **AtoA 调用链 demo** | 60s | 展示 6 Agent 之间如何调用（AtoA 协议可视化） |
| 3 | **飞书原生集成 demo** | 60s | 智能伙伴对话 + 多维表格自动更新 + 妙记转工单 |

### 2.2 录屏脚本模板

```
[0:00-0:30] 开场：商户问题（巴西信用卡拒付率上升）
  → 屏幕：飞书商户群 + 商户发消息截图
  → 旁白："商户面临真实痛点..."

[0:30-1:30] 演示：诊断 Agent 自动响应
  → 屏幕：智能伙伴对话 + 6 Agent 协同面板
  → 旁白："诊断 Agent 自动调取知识/路由/协同..."

[1:30-2:30] 演示：飞书生态闭环
  → 屏幕：多维表格更新 + 妙记自动转工单
  → 旁白："所有协作沉淀在飞书，无需人工传话..."

[2:30-3:00] 收尾：价值回顾
  → 屏幕：3 个核心数字（自助解决率/响应时间/上线周期）
  → 旁白："这就是 OceanMate..."
```

### 2.3 录屏硬约束

| 约束 | 说明 |
|------|------|
| **分辨率** | 1920×1080，最低 1280×720 |
| **格式** | MP4（H.264 编码） |
| **大小** | 单文件 ≤ 100MB（飞书上传限制） |
| **音频** | 旁白必备，可选字幕 |
| **水印** | **不加任何水印**（评分项） |
| **录制工具** | OBS Studio（免费）或 Windows 自带 Xbox Game Bar |

### 2.4 录屏失败预案

| 场景 | 备用方案 |
|------|---------|
| 飞书账号未申请到 | 用模拟数据 + 飞书文档截图 |
| 后端服务挂了 | 提前录好"分段 demo"，拼接 |
| 网络不稳 | 本地起服务 + 内网穿透 / localhost |
| 时间不够 | 至少保 3 分钟主 demo，其他可省 |

## 5. 团队协作 SOP（2 人组）

### 5.1 分工矩阵

| 角色 | 负责 | 产出 |
|------|------|------|
| **A 业务架构师** | Part 1/2 终稿 + 架构图 + 调研 + 飞书配置 | 提交材料 |
| **B Agent 工程师** | 6 Agent 代码 + 飞书 webhook + 录屏 | Demo |

### 5.2 协作节奏

| 时段 | A 在做 | B 在做 |
|------|--------|--------|
| 7-17 晚 | 拍板决策 + 仓库骨架 | 申请飞书账号 |
| 7-18 上午 | Part 1/2 + Mermaid 图 | 6 Agent stub |
| 7-18 下午 | 飞书配置 + 录屏脚本 | 飞书 webhook + 联调 |
| 7-19 上午 | 录屏 + 整理材料 | Bug 修复 + 录屏 |
| 7-19 下午 | cross-check + 提交 | cross-check + 提交 |

### 5.3 冲突解决

| 冲突 | 解决 |
|------|------|
| Part 2 字数超标 | A 砍细节，B 加功能不进 Part 2 |
| Agent 进度滞后 | B 优先保"诊断 Agent + RAG"最小闭环 |
| 飞书账号未到 | A 用 mock 数据写方案，B 用本地服务 demo |
