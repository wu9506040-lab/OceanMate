## 1. 跨境支付行业基本盘

### 1.1 市场规模与增长

| 指标 | 数值 | 来源 | 时间 |
|------|------|------|------|
| 全球跨境支付市场规模 | ~$1.8 万亿 | Statista Cross-Border Payments Report | 2024 |
| 年复合增长率 (CAGR) | ~12% | McKinsey Global Payments Report | 2024-2027 |
| B2C 跨境电商规模 | ~$1.0 万亿 | eMarketer Global Ecommerce | 2024 |
| 数字钱包占比 | 49% | Worldpay Global Payments Report | 2024 |
| 卡支付占比 | 20%（持续下降） | Worldpay | 2024 |
| 先买后付 (BNPL) 增速 | 15% YoY | FIS Global Payments | 2024 |

### 1.2 关键趋势

- **本地化支付偏好**：新兴市场（巴西 PIX、东南亚 GrabPay/GCash、印度 UPI）增速远超卡支付
- **嵌入式金融**：支付公司从"通道"转向"金融服务平台"
- **AI 风控普及**：拒付率上升倒逼 AI 拒付预警（Stripe Radar / Adyen RevenueProtect）
- **监管收紧**：欧盟 PSD2 / 英国 FCA / 美国 Reg E 对支付合规要求提升
- **跨境数据流动合规**：数据本地化要求（如印度 RBI、印尼 PDP Law）

## 2. 头部竞品案例（5 家）

### 2.1 Stripe（Radar 风控 + Sigma 数据）

| 维度 | 内容 |
|------|------|
| AI 能力 | Radar：基于 ML 的拒付预警（VISA/MC 联合训练） |
| 商户自助 | Sigma：商户自助数据分析 |
| 痛点解决 | 主要解决"拒付"，不解决"运营协作" |
| 用户反馈（G2/Capterra） | "风控准但客服响应慢" / "出问题找客服要排 2 天" |
| 借鉴价值 | AI 风控思路可借鉴 |

### 2.2 Adyen（RevenueAccelerate + Uplift）

| 维度 | 内容 |
|------|------|
| AI 能力 | RevenueAccelerate：AI 支付优化（成功率提升）|
| 商户自助 | Uplift：支付方式推荐 |
| 痛点解决 | 主要解决"成功率优化"，不解决"商户侧问答" |
| 用户反馈 | "上线初期不知道选哪些支付方式" / "本地化支付组合推荐不智能" |
| 借鉴价值 | AI 支付方式推荐逻辑 |

### 2.3 Checkout.com（Intelligent Acceptance）

| 维度 | 内容 |
|------|------|
| AI 能力 | Intelligent Acceptance：动态支付路由 |
| 商户自助 | Dashboard + Webhook 状态 |
| 痛点解决 | 主要解决"路由优化"，不解决"客服协同" |
| 用户反馈 | "Webhook 配置复杂" / "出问题要自己看日志" |
| 借鉴价值 | 动态路由算法思路 |

### 2.4 PayPal（Braintree + Riskified 集成）

| 维度 | 内容 |
|------|------|
| AI 能力 | 风控 + 拒付 + 反欺诈 |
| 商户自助 | Resolution Center（自助退单） |
| 痛点解决 | 主要解决"反欺诈"，不解决"跨团队协同" |
| 用户反馈 | "客服电话永远打不通" / "问题升级到工程师要 3-5 天" |
| 借鉴价值 | 自助退单中心的交互设计 |

### 2.5 国内跨境收款（PingPong / Airwallex / WorldFirst）

| 维度 | 内容 |
|------|------|
| AI 能力 | 多为风控类，运营协作类少 |
| 商户自助 | App + 工单系统 |
| 痛点解决 | 主打"费率低/到账快"，运营服务是软肋 |
| 用户反馈（雨果跨境/亿邦动力） | "拒付看不懂" / "对账对不上" / "客服微信回复慢" |
| 借鉴价值 | **国内商户的真实痛点 → 我们的命题方向** |

### 3.3 OP 内部团队视角（推测 + 公开资料）

| 团队 | 痛点 |
|------|------|
| 业务 | 反复给商户解释"该开哪些支付方式"，复用度低 |

## 4. AI Native 商户成功的定义（命题深层理解）

### 4.1 传统模式 vs AI Native

| 维度 | 传统模式（OP 当前） | AI Native（本方案） |
|------|-------------------|---------------------|
| 商户咨询 | 人工客服 / 知识库搜索 | AI 合伙人 + 多 Agent 协同 |
| 问题诊断 | 截图+订单号手工传递 | 自动调取日志+对账+风控规则 |
| 工单分派 | 人工判断归属 | AI 路由 + 自动派单 |
| 知识沉淀 | 客服写 FAQ（滞后） | 对话反哺知识库（实时） |
| 跨团队协同 | 拉群+截图+会议 | 飞书生态原生闭环 |

### 4.2 命题的 3 个真正机会

1. **从"被动响应"到"主动全陪"**：商户问题 AI 主动识别 + 主动跟进
2. **从"传话"到"自动流转"**：6 Agent 自动协同，无需人工拉群
3. **从"知识滞后"到"反哺闭环"**：对话自动转 FAQ，知识库与时俱进

### 4.3 命题的 3 个真正挑战

1. **AtoA 协议**：6 Agent 之间如何稳定调用 / 失败重试 / 状态同步
2. **可解释诊断**：不仅给答案，还要给"根因 + 证据链"
3. **跨境合规**：诊断数据本地化 / 跨境聚合特征

## 7. 调研小结（用于 Part 1 引用的素材包）

### 数据素材（可直接用）
- 全球跨境支付 ~$1.8 万亿，年增 12%（Statista 2024）
- 卡支付占比从 35% → 20%（Worldpay 2024）
- 数字钱包占比 49%（Worldpay 2024）
- BNPL 增速 15% YoY（FIS 2024）

### 竞品素材（可直接用）
- Stripe Radar：风控准但客服响应慢（G2 评价）
- Adyen Uplift：支付方式推荐不智能（Capterra 评价）
- Checkout.com：Webhook 报错归因不清（Trustpilot 评价）
- PayPal：客服电话打不通（Capterra 评价）

### 商户原声（可直接用）
- 巴西拒付率上涨 3 天排查案例
- 巴西 PIX 没开通导致转化率 -30%
- Webhook 回调失败归属不清
- 客服响应慢，反复传截图

### 命题理解（直接进 Part 1）
- 痛点不是"做一个客服机器人"，而是"重构商户成功协作模式"
- AI Native = 主动全陪 + 自动流转 + 反哺闭环
- 机会在 6 Agent + AtoA 协议 + 飞书生态
